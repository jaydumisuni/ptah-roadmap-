## Recovered patterns

- Interface availability is separate from backend availability.
- Desktop and mobile presentations require different layout behavior.
- Operational labels must distinguish planned, configured, connected, running, completed and verified.
- Full operational views can coexist with compact launchers and quick controls.
- Progress, evidence, history and limitations should remain visible.
- The next available action should depend on current evidence rather than previous button clicks.
- Manual handoffs and blocked states deserve first-class presentation.
- Device and application interfaces must show exact target and current state.
- Specialist interfaces should connect through stable bridges rather than importing private internals.
- Settings remain separate from live operational controls.
- Responsive behavior should be built into components instead of accumulated through post-build patch scripts.

## Native Ptah gap

No internal source currently provides the complete neutral Ptah shell with:

- Workspace switching;

- many simultaneous terminal, Browser, Device and Application panels;
- Panel and Layout identities;
- Activity Centre and proof explorer;
- human and automation control handoff;
- responsive desktop, tablet and phone presentations;
- governed public contribution points.
