## Rules confirmed

1. Catalogue inclusion is not donor proof.
2. Profile contribution claims are discovery evidence only.
3. A repository name or README feature claim cannot establish canonical identity, licence, maturity or suitability.
4. Every referenced implementation requires independent source inspection and requirement mapping.
5. Documentation builds are derived Artifacts; Markdown/ADRs/schemas and exact source revisions remain authoritative.
6. Public documentation consumes only allowlisted public sources.
7. Search indexes, generated assets and source maps are checked for private-data leakage.
8. Documentation tooling never becomes Ptah Core, the Human Workspace shell or runtime proof.
9. A missing repository is parked rather than guessed.
10. Cleanup cannot reopen accepted subsystem architecture unless it exposes a concrete unresolved requirement gap.
